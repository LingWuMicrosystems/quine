<objective>
## Goal
Eliminate the remaining code duplication in `related_egraph.rs` (689 lines): the duplicated `insert` (~47 lines) and `union` (~9 lines) methods between `ActionCtx` and `RelatedEGraph`, the duplicated `ActionCtx` construction literal (two 8-line sites), and the near-identical `alloc_id`/`fresh_id` pair. Tighten public API visibility where safe.

## Purpose
After 11-01 (CostTracker → cost.rs) and 11-02 (ReverseIndex → reverse_index.rs), the remaining duplication in `related_egraph.rs` is structural: `RelatedEGraph` has public `insert`, `union`, and `fresh_id` methods that contain the exact same logic as their `ActionCtx` counterparts. The pattern established in `RelatedEGraph::apply_action` — construct an `ActionCtx` and delegate — should be applied consistently instead of duplicating logic. This reduces ~65 lines, eliminates 4 duplicated code blocks, and makes `ActionCtx` the single source of truth for mutation logic.

## Output
- `related_egraph.rs`: ~65 lines removed (689 → ~625), all duplication eliminated
- `RelatedEGraph::insert`, `union`, `fresh_id` now delegate to `ActionCtx` via `action_ctx()` helper
- `action_ctx()` helper method added, replacing two identical `ActionCtx` construction literals
- API tightened: private where possible, `pub(crate)` where crate-internal
</objective>
