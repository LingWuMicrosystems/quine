<objective>
## Goal
Extract reverse_index tracking into a new `quine-core/src/reverse_index.rs` module: create `ReverseIndex` struct wrapping `Map<Value, Set<(TableId, RowIndex)>>` with `insert`, `merge`, `remove`, and `get` methods. Eliminate the duplicated insert (ActionCtx + RelatedEGraph) and merge (ActionCtx + RelatedEGraph + rebuild) patterns.

## Purpose
`related_egraph.rs` is 721 lines after the 11-01 CostTracker extraction. The reverse_index is a coherent, self-contained concern — a `Map<Value, Set<(TableId, RowIndex)>>` with well-defined operations (insert, remove, merge, query). The same insert/entry+or_default+insert pattern appears twice, and the same merge/remove+extend pattern appears three times. Extracting it reduces `related_egraph.rs` by ~60 lines, eliminates the pattern duplication, and follows the CostTracker extraction pattern established in 11-01.

## Output
- New file: `quine-core/src/reverse_index.rs` (~70 lines) with `ReverseIndex` struct + impl
- `related_egraph.rs`: ~60 lines removed, replaced with delegation to `ReverseIndex` + pattern calls
- `lib.rs`: `pub mod reverse_index;` added
- 10 existing reverse_index integration tests continue to pass unchanged (they use `eclass_enodes` public API)
</objective>
