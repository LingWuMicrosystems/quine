<objective>
## Goal
Extract cost tracking logic into a new `quine-core/src/cost.rs` module: create `CostTracker` struct holding `cost_models`, `eclass_cost`, and `cost_select`, with `compute_and_update_eclass_cost` and `merge_eclass_cost_into` as methods. Eliminate the duplicated method/function pairs in `related_egraph.rs`.

## Purpose
`related_egraph.rs` is 892 lines and growing. The cost subsystem (cost lattice, cost models, eclass cost tracking, cost_select) is a coherent, self-contained concern with clear boundaries — it takes tables + union_find as read-only inputs and maintains its own mutable state. Extracting it reduces `related_egraph.rs` by ~150 lines, eliminates the ActionCtx/standalone duplication, and establishes the pattern for further extraction (reverse_index, query engine).

## Output
- New file: `quine-core/src/cost.rs` (~130 lines) with `CostTracker` struct + impl
- `related_egraph.rs`: ~150 lines removed, replaced with delegation to `CostTracker`
- `lib.rs`: `pub mod cost;` added
- 5 lattice property tests moved from `related_egraph.rs` → `cost.rs`
</objective>
