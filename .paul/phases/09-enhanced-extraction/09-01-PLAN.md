<objective>
## Goal
Harden the ILP extraction pipeline by wiring unused `ILPConfig` fields (`time_limit_ms`, `max_cse_edges_warning`), adding end-to-end integration tests for the `extract optimal` DSL path, and creating fuzz tests that compare ILP results against brute-force enumeration for small random e-graphs.

## Purpose
Phase 8 delivered a working `extract optimal` pipeline but left `ILPConfig` fields as placeholders and had no integration-level verification. Phase 9 closes these gaps — making the solver configurable, proving the full DSL→CLI→solver→output chain works, and building confidence in the B&B-CR solver's correctness through randomized testing.

## Output
- Functional `time_limit_ms` (as B&B node budget) and `max_cse_edges_warning` (user-visible warning)
- Integration test exercising `extract optimal` end-to-end via CLI
- Fuzz test framework generating random e-graphs and verifying ILP optimality against brute force
</objective>

## AC-1: time_limit_ms enforces B&B search budget
```gherkin
Given an ILPConfig with time_limit_ms = Some(10) (mapped to ~10,000 node budget)

Then  the B&B search terminates early after exploring at most 10,000 nodes
      AND the result has optimal = false
      AND nodes_explored ≤ 10,000
```

```gherkin
      AND CLI prints the warning to stderr

```

    - Convert `time_limit_ms` to `max_nodes: Option<u64>`: heuristic `ms * 1000` nodes/ms, pass to branch_and_bound

    **quine-solver/src/solver.rs — B&B early termination:**
    - Add `max_nodes: Option<u64>` parameter to `branch_and_bound()` signature
    - After `stats.nodes_explored += 1`: check `if max_nodes.map_or(false, |m| stats.nodes_explored >= m) { return; }`
    - This returns without updating `best` (the incumbent), so the caller sees `optimal = false` when budget exhausted
    - Update existing solver unit tests to pass `max_nodes: None`

    **quine-cli/src/main.rs — Print warnings:**
    - After `ilp_extract()` call (both file and REPL paths): check `result.warning` and print to stderr via `eprintln!`
    - Keep existing `ILPConfig::default()` — defaults already wired (time_limit_ms=Some(1000), max_cse_edges_warning=50)

    Avoid: changing ILPConfig field names (API stability), using std::time in solver (no_std constraint)
