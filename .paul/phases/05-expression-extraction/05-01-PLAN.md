<objective>
## Goal
Implement cost-aware expression extraction: when `extract <expr>` is evaluated, resolve the expression to an eclass in the e-graph, use `cost_select` to find the cheapest equivalent enode, materialize it to a `Term`, and print the result.

## Purpose
Phase 4 gave us `extract <expr>` syntax and `CompiledUnit::Extract(Expr)`. Phase 5 makes it actually work — evaluating the expression against the e-graph, using the cost lattice (built in Phase 3) to find the cheapest equivalent, and materializing the result. This closes the v0.2 milestone by delivering the end-to-end `extract` feature.

## Output
- `Table::get_by_key()` — public key lookup on tables
- `EngineContext::evaluate_expr()` — resolves `Expr` to canonical eclass `Value`
- `EngineContext::materialize_cheapest()` — cost-aware term materialization from an eclass
- `CompiledUnit::Extract` arm in `apply()` wired to evaluate + materialize
- CLI handles `Command::Extract` specially, prints the resulting `Term`
- Integration tests with cost models and multiple equivalent forms
</objective>

    2. **Add compile-time validation for Extract** (`quine-frontend/src/compile/mod.rs` lines 103-105):
       Replace the current no-validation pass-through with validation:
       - Check that the expression contains no `Variable` (walk the Expr tree)
       - For `FunctionCall(name, _)`: validate that `name` exists in `table_types.name_map` OR can be resolved via `data_types.cons2type_map` (constructor lookup)
       - Return `CompileError::UnknownConstructor` or a new `CompileError::VariableInExtract` variant as appropriate
       - Add the new error variant(s) to `quine-frontend/src/error.rs`

    3. **Add `evaluate_expr()` to `EngineContext`** (`quine-frontend/src/lib.rs`):
       Add a method that resolves an `Expr` to a canonical eclass `Value`:
       ```rust
       pub fn evaluate_expr(&self, expr: &Expr) -> Result<Value, String>
       ```
       - `Expr::AtomOrVariable(AtomOrVariable::Atom(atom))`: convert via `atom_to_value(atom.clone(), &mut self.interner.clone())` then `self.regraph.find(value)`. Note: `atom_to_value` takes `&mut Interner` for string interning; for evaluation (not compilation), use a temporary clone or pass a dummy interner since strings should already be interned from compilation.
       - `Expr::FunctionCall(FunctionCall(name, args))`: 
         - Resolve `name` to `TableId` via `self.table_types.name_map`. If not found directly, try resolving via `data_types.cons2type_map` (search for keys ending with `.name`).
         - Recursively evaluate each arg to a canonical `Value`.
         - Build key `Row` from evaluated arg values (first N columns = arity).
         - Look up via `table.get_by_key(&key)` — if found, get the value column (at index `arity`) and canonicalize.
         - If not found, return an error.
       - `Expr::AtomOrVariable(AtomOrVariable::Variable(_))`: return error (should be caught at compile time, but defensive).

    4. **Add `materialize_cheapest()` to `EngineContext`** (`quine-frontend/src/lib.rs`):
       Add a cost-aware materialization method:
       ```rust
       pub fn materialize_cheapest(&self, eclass: Value) -> Term
       ```
       - Canonicalize `eclass` via `self.regraph.find(eclass)`.
       - If `cost_select` has an entry for the canonical eclass, use that `(tid, row_idx)` as the starting point.
       - Otherwise fall back to `extract_inner` (uses `find_defining_row`).
       - Recursively materialize children using the same cost-aware strategy: for each child eclass, use `cost_select` to pick the cheapest enode, then materialize from it.
       - Use a `&mut Set<Value>` visited set (like `extract_inner`) to detect cycles → `Term::Cyclic`.
       - For literal-typed children, convert to `Term::Literal` via `atom_from_value`.

    5. **Wire `CompiledUnit::Extract` in `apply()`** (`quine-frontend/src/lib.rs` lines 87-89):
       Fill in the no-op:
       ```rust
       CompiledUnit::Extract(expr) => {
           self.last_extract = Some(self.evaluate_and_extract(&expr));
       }
       ```
       Add `last_extract: Option<Term>` field to `EngineContext` (default `None`). The CLI reads this to print the result.
       
       Add `evaluate_and_extract(&self, expr: &Expr) -> Result<Term, String>` convenience method that chains `evaluate_expr` → `materialize_cheapest`.

    Avoid: changing `extract()` or `extract_inner()` — the existing methods are used by query result printing and should keep their current behavior (scan-based, not cost-aware). The new methods are additive.
