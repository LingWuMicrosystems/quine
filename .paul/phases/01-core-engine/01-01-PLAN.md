<objective>
## Goal
Implement the `reverse_index` (eclass → enode references) and `eclass_enodes` query method on `RelatedEGraph`, enabling reverse lookup from any canonical eclass to all table rows that reference it.

## Purpose
The reverse index is a core e-graph data structure that enables efficient lookup of which enodes belong to an eclass. This is essential for e-graph extraction (finding the "best" enode for an eclass), debugging, and future features like pattern matching against eclasses.

## Output
- Populated `reverse_index` in `RelatedEGraph` (maintained through insert, union, and rebuild)
- Working `eclass_enodes(&self, eclass: Value) -> Set<(TableId, RowIndex)>` method
- 12 passing integration tests in `quine-core/tests/reverse_index.rs`
</objective>


```gherkin

Then reverse_index does not contain entries for those values
```

## AC-6: Canonical changes update reverse_index keys

## AC-7: eclass_enodes returns all enodes for an eclass

<task type="auto">
  <name>Task 2: Maintain reverse_index during rebuild and implement eclass_enodes</name>
  <files>quine-core/src/related_egraph.rs</files>
  <action>
    **2a. Track scanned row index in rebuild:**
    Change the `pairs` collection in `rebuild()` to preserve the scanned row index:
    From: `.flat_map(|&idx| rebuild_row(table, idx, &self.union_find))`
    To:   `.filter_map(|&idx| rebuild_row(table, idx, &self.union_find).map(|r| (idx, r)))`
    This yields `(scanned_idx, (existing_idx, old, new))` — needed to remove merged rows from reverse_index.

    **2b. Remove merged rows from reverse_index during rebuild:**
    When processing each pair `(scanned_idx, (existing_idx, old, new))`:
    - The row at `scanned_idx` is being absorbed into `existing_idx`
    - For the value column of the scanned row, remove `(table_id, scanned_idx)` from reverse_index
    - Canonicalize the value first: `let scanned_value = self.tables[tid].get_all_row(scanned_idx).0[arity]`
    - `let canonical = self.union_find.find(scanned_value)`
    - `self.reverse_index.entry(canonical).and_modify(|s| { s.remove(&(tid, scanned_idx)); });`

    **2c. Merge reverse_index entries on union:**
    When `union(old, new)` produces a pending union `(parent, child)` in rebuild, also merge reverse_index:
    - After processing pairs where values were unioned (no merge fn), take the entries from `child`'s reverse_index set and move them to `parent`'s set
    - Remove `child`'s entry from reverse_index after merging
    - This should happen alongside where `self.pending_unions.extend(new_pairs)` is called

    **2d. Implement eclass_enodes method:**
    ```rust
    pub fn eclass_enodes(&self, eclass: Value) -> Set<(TableId, RowIndex)> {
        let canonical = self.union_find.find(eclass);
        self.reverse_index.get(&canonical).cloned().unwrap_or_default()
    }
    ```
    - Canonicalize the input via `union_find.find`
    - Lookup in reverse_index
    - Return cloned set or empty set

    Avoid: leaving stale entries in reverse_index after row merge or canonical change.
  </action>
