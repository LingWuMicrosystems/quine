<objective>
## Goal
Add `extract optimal <expr>` DSL syntax that routes through the B&B-CR ILP solver, and fix the `build_extraction_dag` empty-eclass bug (#18) so the solver works with any e-graph shape.

## Purpose
Phase 7 delivered a fully-tested ILP solver (28 tests, 0 failures). Phase 8 connects it to the user-facing extraction pipeline so `extract optimal <expr>` produces globally optimal (not just greedy) extractions when CSE edges exist.

## Output
- `extract optimal <expr>` parses, compiles, and produces ILP-optimal extraction via `ilp_extract()`
- `extract <expr>` (without `optimal`) continues to use greedy DP — backward compatible
- `build_extraction_dag` no longer produces eclasses with 0 enodes
- quine-cli gains quine-solver dependency
</objective>

    - Change `extract_query` rule from `{ "extract" ~ expr }` to `{ "extract" ~ "optimal"? ~ expr }`

    - Add `ExtractMode` enum: `Greedy` (default), `Optimal`
    - Change `Extract(Expr)` to `Extract(Expr, ExtractMode)` in Command enum

    **3. Parser** (`quine-cli/src/pest_parser.rs`):
    - In `parse_command`, match `extract_query`: check if inner contains "optimal" keyword, parse accordingly
    - Map to `Command::Extract(expr, mode)` where mode is `Optimal` if keyword present, else `Greedy`

    **4. Compilation** (`quine-frontend/src/compile/mod.rs`):
    - `CompiledUnit::Extract(Expr)` → `CompiledUnit::Extract(Expr, ExtractMode)`
    - Thread through the `ExtractMode` in the match arm

    **5. EngineContext** (`quine-frontend/src/lib.rs`):
    - `CompiledUnit::Extract(expr, mode)` match arm: store mode alongside result
    - Add `last_extract_mode: Option<ExtractMode>` field (or just store the mode)
    - For now, Greedy path calls `evaluate_and_extract` as before; Optimal path stores the expr for CLI to handle
    - **Key decision:** quine-frontend cannot call quine-solver (circular dep). Instead, expose `last_extract_expr: Option<Expr>` and `last_extract_mode: Option<ExtractMode>` so the CLI can call `ilp_extract`.
    - Add field: `pub last_extract_info: Option<(Expr, ExtractMode)>` — populated on Extract, consumed by CLI

    Avoid: importing quine-solver in quine-frontend (circular dependency). Keep quine-frontend solver-agnostic.
    Avoid: breaking existing `extract <expr>` behavior — the Greedy path must be identical to pre-Phase-8.

<task type="auto">
  <name>Task 3: Fix #18 — Filter empty eclasses in build_extraction_dag</name>
  <files>quine-solver/src/dag.rs</files>
  <action>
    **Fix `build_extraction_dag`** (`quine-solver/src/dag.rs`):

    In the BFS loop (Pass 1, lines 96-147), after gathering enodes with `eclass_enodes()`:
    - If the enodes list is empty, **skip this eclass** — do not push it into `eclasses` vector, do not add to `eclass_map`
    - This requires restructuring: gather enodes, check if empty, if so continue without incrementing/pushing
    - Children of empty eclasses: since the empty eclass is skipped, child edges recorded in `child_parents` from other enodes that reference it should either be: (a) also skipped, or (b) filtered in Pass 2
    - **Simplest correct approach:** Before the BFS, pre-compute which eclasses have enodes (validate each queued eclass). In the BFS, when enqueuing a child, verify it has enodes. Skip child eclasses with 0 enodes — treat them as non-existent (no edge recorded, no CSE detection).
    - Specifically: at lines 135-138, before enqueuing a child, check `regraph.eclass_enodes(child_canon).next().is_some()`. If the child eclass has no enodes, don't record the parent→child edge and don't enqueue.

    **Edge case — root eclass with 0 enodes:**
    - `ilp_extract` (lib.rs line 73) already handles empty `dag.eclasses` — returns early with `term: None`. No change needed there.

    Avoid: index remapping (complex, error-prone). The pre-check approach at child enqueue time is simpler and correct.
    Avoid: breaking existing tests that use dual-table schema — they should still pass since Leaf tables have arity 0 (no eclass-typed children to trigger the bug).
  </action>
  <verify>cargo test -p quine-solver 2>&1 | tail -5</verify>
