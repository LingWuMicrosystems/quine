<objective>
## Goal
Create the `quine-solver` crate (`#![no_std]` + `extern crate alloc`) with its public API surface, type predicate helpers, and the `ExtractionDAG` builder that snapshots e-graph state into an owned data structure for the solver.

## Purpose
This is the foundation for Plans 07-02 (solver algorithm) and 07-03 (tests). The `ExtractionDAG` is the bridge between `RelatedEGraph` and the solver — it freezes e-graph state into a `Vec`-backed DAG that the B&B-CR algorithm operates on without shared references. All integration uses existing public APIs; zero changes to `quine-core` internals.

## Output
- `quine-solver/Cargo.toml` — crate manifest (depends on quine-core, quine-frontend)
- `quine-solver/src/lib.rs` — `#![no_std]` + public types: `ILPConfig`, `ILPResult`, `ilp_extract()` stub

- `quine-solver/src/dag.rs` — `ExtractionDAG`, `EclassNode`, `CseEdge`, `build_extraction_dag()`
- `Cargo.toml` (workspace root) — add `quine-solver` to members and workspace dependencies

<task type="auto">
  <name>Task 3: Build dag.rs — ExtractionDAG builder</name>
  <files>quine-solver/src/dag.rs</files>
  <action>
    Implement the ExtractionDAG data structure and builder per design report §5.1 pseudocode:

    1. **Data types:**
       - `pub struct EclassNode { canonical: Value, enodes: Vec<(TableId, RowIndex)> }`
         (enodes stored as (table_id, row_index) pairs — all needed to look up children/costs)
       - `pub struct CseEdge { pub child_eclass: usize, pub parent_enodes: Vec<(usize, usize)> }`
         (child_eclass = index into eclasses vec; parent_enodes = list of (eclass_idx, enode_idx) that reference it)
       - `pub struct ExtractionDAG { pub eclasses: Vec<EclassNode>, pub root: usize, pub cse_edges: Vec<CseEdge> }`

    2. **`pub fn build_extraction_dag(regraph: &RelatedEGraph, root_eclass: Value) -> ExtractionDAG`:**
       - Canonicalize root: `let root = regraph.find(root_eclass);`
       - BFS from root, building eclass_map: `Map<Value, usize>` (canonical Value → array index)
       - For each eclass, fetch enodes via `regraph.eclass_enodes(canonical)`
       - Enumerate children: for each enode (tid, ridx), get table via `regraph.get_table(tid)`, get row via `table.get_all_row(ridx)`, iterate columns — if `type_is_eclass(col_type)`, canonicalize child via `regraph.find(child_val)` and enqueue if not visited
       - BFS order is topological for DAG (acyclic by construction) — eclasses are stored in discovery order
       - After BFS, second pass to identify CSE edges: build `child_parents: Map<usize, Vec<(usize, usize)>>` mapping each child eclass index to its parent (eclass_idx, enode_idx) pairs; any child with >1 parent is a CSE edge
       - Populate `cse_edges` from `child_parents` entries where `parents.len() > 1`

    3. Edge cases:
       - Empty root (no enodes in root eclass): return ExtractionDAG with empty eclasses
       - Cyclic check: assert no eclass visited twice in BFS (DAG invariant — safety assertion)

    Avoid: Cloning the entire e-graph — only extract what's reachable from root. Avoid: depending on solver.rs types (dag.rs is a leaf dependency). Avoid: using HashMap (not in alloc) — use BTreeMap from alloc::collections for eclass_map.
  </action>
