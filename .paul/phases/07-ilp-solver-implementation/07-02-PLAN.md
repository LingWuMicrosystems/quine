<objective>
## Goal
Implement the Branch-and-Bound with Combinatorial Relaxation (B&B-CR) algorithm from design report §4.4 — the core ILP solver for cost-optimal expression extraction.

## Purpose
This is the heart of the v0.3 ILP Solver milestone. Without CSE-aware optimization, `materialize_cheapest` overcounts shared subexpression costs, producing suboptimal extractions when eclasses are referenced by multiple parents. B&B-CR guarantees the global cost-optimum by branching on CSE ownership decisions with a fast DAG shortest-path relaxation at each node.

<task type="auto">
  <name>Implement combinatorial relaxation module</name>
  <files>quine-solver/src/relaxation.rs</files>
  <action>
    Create `quine-solver/src/relaxation.rs` with:

    **Types:**
    - `FixedDecision` enum:
      - `Selected(TableId, RowIndex)` — force a specific enode in this eclass
      - `OwnedBy(usize)` — this eclass's cost is owned by parent_idx (for CSE accounting)
      - `NotShared` — break CSE coupling; each parent independently pays this eclass's cost
    - `RelaxedSolution` struct:
      - `enode_selection: Vec<Option<(TableId, RowIndex)>>` — indexed by eclass_idx
      - `cost: u64` — total cost from root

    **Functions (following design report §4.4 pseudocode):**

    1. `solve_relaxation(dag, regraph, fixed) -> RelaxedSolution`
       - Iterate eclasses in reverse (leaves-first: `(0..len).rev()`)
       - For each eclass: if present in `fixed`, call `apply_fixed()`; otherwise:
         - For each enode: cost = constructor_cost + sum(child_costs with CSE adjustment)
         - CSE adjustment: if child has `FixedDecision::OwnedBy(parent)` where parent ≠ current, add 0; otherwise add cost[child]
         - Pick cheapest enode (min cost)
       - Return RelaxedSolution with total cost at `dag.root`

    2. `apply_fixed(dag, regraph, fixed, eclass_idx) -> (u64, Option<(TableId, RowIndex)>)`
       - Match FixedDecision:
         - `Selected(tid, ridx)`: compute that enode's cost (constructor + children with CSE adjustment), return (cost, Some((tid, ridx)))
         - `OwnedBy(_)`: this eclass is a CSE child being "owned" — still pick cheapest enode normally (ownership affects parent accounting, not child selection)
         - `NotShared`: pick cheapest enode normally (cost counted independently at each parent — this IS the relaxation default)

    3. `find_cse_violations(dag, relaxed) -> Vec<(usize, usize)>`  // (cse_edge_index, selected_parent_count)
       - For each CseEdge: count how many parent enodes are selected in `relaxed.enode_selection`
       - Return edges where selected_parents > 1 (double-counting would occur)

    4. `pick_branching_eclass(dag, violations) -> usize`
       - Heuristic: pick the eclass with the most selected parents (largest CSE overcount)
       - Ties broken by topological_order (deeper first)

    5. `solve_dag_shortest_path(dag, regraph) -> Solution`
       - Fast path when `dag.cse_edges.is_empty()` — no B&B needed
       - Runs `solve_relaxation()` with empty fixed map
       - Converts RelaxedSolution to Solution

    **Important:**
    - CSE adjustment logic is the core of the relaxation. When computing enode cost for eclass `p`:
      - For each child eclass `c` of the enode:
        - If `fixed` contains `c` with `OwnedBy(owner)` and `owner != p`: cost += 0 (child cost counted at owner)
        - Else: cost += cost[c] (default: each parent pays child cost — this is the "double-count" that B&B branches to resolve)
    - Topological order: iterate `(0..dag.eclasses.len()).rev()` (BFS order reversed = leaves first since root is index 0)

<task type="auto">
  <name>Implement B&B framework + solution extraction + wire into lib.rs</name>
  <files>quine-solver/src/solver.rs, quine-solver/src/lib.rs</files>
  <action>
    Create `quine-solver/src/solver.rs` with:

    **Types:**
    - `Solution` struct (from design report §4.4):
      - `enode_selection: Vec<Option<(TableId, RowIndex)>>` — indexed by eclass_idx
      - `cost: u64`

    - `BnBNode` struct:
      - `fixed: BTreeMap<usize, FixedDecision>` — accumulated decisions (clone-on-branch)
      - No parent pointer needed — recursion uses value semantics (clone map, pass to child)

    **Functions (following design report §4.4 pseudocode):**

    1. `branch_and_bound(dag, regraph, node, best, stats)`
       - Solve relaxation at this node: `solve_relaxation(dag, regraph, node.fixed)`
       - Bound: if `relaxed.cost >= best.cost`, return (prune)
       - Check feasibility: `find_cse_violations(dag, relaxed)`
       - If no violations: update `best` with relaxed solution, return (feasible leaf)
       - Branch: pick eclass from violations
         - **Branch A (NotShared):** clone node, fix eclass as NotShared, recurse
         - **Branch B (OwnedBy):** for each parent in cse_edge.parent_enodes:
           - Clone node, fix eclass as OwnedBy(parent_idx), fix parent as Selected(enode_idx)
           - Recurse
       - Stats: increment `nodes_explored` at each node

    2. `extract_solution_from_dag(dag, regraph, solution) -> Term`
       - Following design report §5.5 pseudocode
       - Build eclass_map: `BTreeMap<Value, usize>` from `dag.eclasses.iter().enumerate()`
       - Call recursive `build_term()` starting from `dag.root`
       - `build_term(eclass_idx, selection, visited)`:
         - Guard: if eclass_idx in visited → return `Term::Literal(Atom::U64(canonical.0))` (cycle safety)
         - If `selection[eclass_idx]` is None → return `Term::Literal(Atom::U64(canonical.0))`
         - If Some((tid, ridx)): look up table/row, recurse on eclass-typed children via `atom_from_value` for literals
         - Return `Term::App(table_name, children)`
       - Note: `atom_from_value`, `type_is_eclass` from formulation.rs; `regraph.find()` for canonical child values

    **Update `quine-solver/src/lib.rs`:**
    - Add `pub mod relaxation;` and `pub mod solver;` declarations
    - Replace `ilp_extract()` stub with real implementation:
      ```
      1. Build ExtractionDAG via build_extraction_dag(regraph, root_eclass)
      2. If dag.eclasses.is_empty(): return empty ILPResult
      3. Fallback check: if dag.eclasses.len() > config.max_eclasses → greedy fallback
      4. If dag.cse_edges.is_empty(): fast path via solve_dag_shortest_path()
      5. Else: initialize best = Solution { cost: u64::MAX, ... }, stats, run branch_and_bound()
      6. Extract term via extract_solution_from_dag()
      7. Return ILPResult { term, optimal, nodes_explored, cost }
      ```
    - `optimal` is true unless fallback was used (greedy fallback ≠ guaranteed optimum in the presence of CSE)

    **Important:**
    - B&B recursion depth bounded by number of CSE edges (typically < 50)
    - Each branch clones the fixed map — BTreeMap clone is O(|fixed|), bounded by CSE count
    - No BinaryHeap needed — depth-first B&B is fine for initial implementation (design report says "basic" for Phase 7)
    - Constructor costs read via `constructor_cost(regraph, table_name)` from formulation.rs
    - Solution extraction uses existing public APIs on RelatedEGraph — no internal access needed
    - eclass_map for solution extraction built on-the-fly from eclasses vector (avoids modifying dag.rs)
