<objective>
## Goal
Validate the B&B-CR solver with comprehensive tests: unit tests for each function, integration tests against real e-graphs (worked examples from the design report), exhaustive brute-force verification for small instances, and property-based invariant checks.

## Purpose
The solver was verified by compilation only in 07-02. Before integrating into the pipeline (Phase 8), it needs runtime tests proving correctness for all scenarios: no-CSE fast path, CSE double-counting, nested CSE after #16 fix, bound pruning, threshold fallback, and edge cases.

## Output
- `#[cfg(test)]` modules in `relaxation.rs` and `solver.rs` with unit tests
- `quine-solver/tests/scenarios.rs` — worked examples 8.1, 8.2, nested CSE
- `quine-solver/tests/exhaustive_verify.rs` — brute-force enumeration on small random e-graphs
- `quine-solver/tests/property_tests.rs` — invariants (cost bounds, validity, edge cases)
</objective>

  <action>
    Add `#[cfg(test)] mod tests` blocks to relaxation.rs and solver.rs with unit tests
    that exercise individual functions with programmatically constructed ExtractionDAGs
    (no RelatedEGraph dependency — build DAG structs directly):

    **relaxation.rs tests:**
    - `test_solve_relaxation_tree` — 3-eclass tree DAG (no CSE edges), verify cost = sum of cheapest enodes
    - `test_solve_relaxation_cse_ownedby` — DAG with 1 CSE edge, fix child as OwnedBy(parent0), verify parent1 gets child cost = 0
    - `test_solve_relaxation_notshared` — DAG with CSE, fix child as NotShared, verify both parents pay child cost
    - `test_solve_relaxation_selected` — fix eclass to specific enode, verify that enode is selected
    - `test_enode_cost_with_cse` — verify CSE adjustment: OwnedBy(owner≠parent) → child cost not added
    - `test_find_cse_violations_none` — relaxed solution with no double-counting → empty violations
    - `test_find_cse_violations_found` — relaxed solution with >1 parent selected → violation returned
    - `test_pick_branching_eclass` — two violations, verify picks one with most selected parents
    - `test_solve_dag_shortest_path` — tree DAG → Solution with correct cost
    - `test_fixeddecision_merge` — verify Default + field assignment: selected + cse coexist on same eclass (#16 regression)

    **solver.rs tests:**
    - `test_branch_and_bound_tree` — no CSE edges → single-node B&B, cost = relaxation cost, stats.nodes_explored = 1
    - `test_branch_and_bound_single_cse` — 1 CSE edge (design report §8.1 shape) → B&B finds optimal, cost < relaxation
    - `test_branch_and_bound_nested_cse` — DAG with 2 CSE edges sharing same child → exercises #16 fix (OwnedBy + Selected coexist), cost correct
    - `test_branch_and_bound_pruning` — setup where Branch A relaxation ≥ incumbent → verify Branch A pruned (relaxation never called for that branch, or cost stays at incumbent)
    - `test_extract_solution_from_dag` — known Solution → valid Term with correct structure
    - `test_build_term_cycle_guard` — DAG with self-referencing eclass → Term::Literal emitted instead of infinite recursion

    **dag.rs changes (if needed):**
    - If `EclassNode`, `CseEdge`, or `ExtractionDAG` need `PartialEq` for test assertions, add the derive.
      Check existing derives first — they already have `Debug, Clone`.

    **Avoid:**
    - Do NOT change any production logic — tests only
    - Do NOT add new public APIs unless essential for testability (e.g., a test-only constructor for ExtractionDAG)
    - Use minimal DAGs: 3-5 eclasses, 1-2 enodes each, directly constructed
    - No RelatedEGraph in unit tests — build ExtractionDAG structs manually
  </action>

<task type="auto">
  <name>Task 2: Integration tests — worked examples, exhaustive verification, property invariants</name>
  <files>quine-solver/tests/scenarios.rs, quine-solver/tests/exhaustive_verify.rs, quine-solver/tests/property_tests.rs</files>
  <action>
    Create three integration test files in `quine-solver/tests/`. Integration tests use std
    and construct real `RelatedEGraph` instances, then call `ilp_extract()`.

    **tests/scenarios.rs — Worked Examples:**
    - `test_example_8_1_cse_double_counting` — construct e-graph matching design report §8.1:
      - Table `Expr` with constructors Add(Mul2, Mul2), Mul2(Const), Const(u64)
      - Costs: Add=10, Mul2=5, Const=1
      - Create e-graph state where eclass D (Const(1)) is shared by B and C (Mul2 nodes)
      - Verify: `ilp_extract().optimal == true`, `cost == 21`, greedy_cost ≥ 21
    - `test_example_8_2_cost_tradeoff` — construct e-graph matching design report §8.2:
      - Table `Op` with Mul(Op,Op), Add(Op,Op), Val(u64), Square(Op)
      - Costs: Mul=20, Add=10, Val=1, Square=5
      - Create e-graph with V2 shared by M1(Mul) and M2(Square)
      - Verify: `ilp_extract().optimal == true`, `cost == 37`
    - `test_nested_cse` — construct e-graph with 2 CSE edges sharing same child:
      - 3 parents referencing child eclass → 2 CSE edges
      - Verify the #16 fix: solver doesn't crash, optimal result found

    **tests/exhaustive_verify.rs — Brute-Force Correctness:**
    - `test_exhaustive_small_random` — generate 20+ random small e-graphs:
      - 3-8 eclasses, 1-3 enodes each, 0-3 CSE edges
      - Write a brute-force enumerator that tries every combination of enode selections
        and computes true minimum cost with CSE-aware accounting
      - Compare ilp_extract().cost == brute_force_minimum for every instance
      - If any mismatch, panic with the DAG structure for debugging
    - `test_exhaustive_known_cases` — hand-crafted minimal edge cases:
      - 1 eclass, 1 enode → trivial
      - 1 eclass, 2 enodes different costs → picks cheaper
      - Tree of 3 eclasses → optimal matches greedy

    **tests/property_tests.rs — Invariants:**
    - `test_ilp_cost_le_greedy` — for any e-graph with CSE: `ilp.cost ≤ greedy.cost`
    - `test_no_cse_optimal` — when CSE edges = 0: `ilp.optimal == true` AND `ilp.cost == greedy.cost`
    - `test_empty_dag_returns_none` — `ilp_extract()` with nonexistent root → `ILPResult.term == None`
    - `test_max_eclasses_fallback` — DAG with eclasses > max_eclasses → `optimal == false`, `nodes_explored == 0`
    - `test_extracted_term_valid` — walk the extracted Term tree, verify no dangling U64 literals where eclass references expected
    - `test_cost_non_negative` — verify `ilp.cost < u64::MAX` (not overflowed)
    - `test_single_eclass` — root = only eclass → picks cheapest enode, optimal = true

    **Avoid:**
    - Do NOT test quine-cli — linker issues with -liconv on macOS. All tests are library-only.
    - Do NOT add quine-cli to workspace build verification.
    - Keep test e-graphs small (≤10 eclasses) for fast execution.
    - Use deterministic seeding for random generation in exhaustive tests.
  </action>
