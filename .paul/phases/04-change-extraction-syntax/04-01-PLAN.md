<objective>
## Goal
Change the extraction DSL syntax from `extract <pattern> print(<vars>)` to `extract <expr>`, where `<expr>` is a concrete value expression (constructor calls with literals, no pattern variables).

## Purpose
The old syntax (`extract <pattern> print(<vars>)`) was designed for pattern matching with variable binding — the same head/guard/leteq patterns used by `query` and `rule`. However, extraction doesn't need pattern matching; it needs a concrete expression to find in the e-graph and return the lowest-cost equivalent. The new syntax `extract <expr>` is simpler, more intuitive, and matches the actual semantics.

## Output
- Grammar rule `extract_query` simplified to `"extract" ~ expr`
- `Command::Extract` carries `Expr` instead of `(Heads, Vec<String>)`
- `CompiledUnit::Extract` carries `Expr` instead of `(rule::Query, Vec<String>)`
- Parser extracts an `expr` instead of heads + variables
- No `heads2query` compilation for extract — the `Expr` is stored directly
- All 6 extract tests updated to new syntax
- README extract section updated with new syntax examples
</objective>

    1. **Grammar** (`docs/grammar.pest` line 39):
       Replace:
       ```
       extract_query = { "extract" ~ heads ~ "print" ~ "(" ~ variable ~ ("," ~ variable)* ~ ")" }
       ```
       With:
       ```
       extract_query = { "extract" ~ expr }
       ```
       `expr` is already defined as `{ function_call | atom_or_variable }` — no new rules needed.

    2. **Parser** (`quine-cli/src/pest_parser.rs` lines 304-309):
       Replace the extract_query arm in `parse_command()`:
       ```rust
       Rule::extract_query => {
           let mut parts = inner.into_inner();
           let heads = parse_heads(parts.next().unwrap());
           let vars: Vec<_> = parts.map(|p| parse_variable(p)).collect();
           Command::Extract(heads, vars)
       }
       ```
       With:
       ```rust
       Rule::extract_query => {
           let expr = parse_expr(inner.into_inner().next().unwrap());
           Command::Extract(expr)
       }
       ```

       Update the Display impl (lines 333-349) from the heads+vars format to:

