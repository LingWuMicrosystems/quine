## Key Decisions (this session)

- **Cost lattice:** `L = (u64, ⊑, ⊥, ⊤, ⊔)` where `a ⊑ b iff a ≥ b`, `a ⊔ b = min(a, b)`, `⊥ = u64::MAX`, `⊤ = 0`, `+ = saturating_add`
- **Incremental maintenance:** Costs computed eagerly at `insert` (enode cost), merged eagerly at `union` (min + inherit cost_select), redirected at `rebuild` (absorbed → surviving RowIndex)
- **cost_select tracking:** `Map<Value, (TableId, RowIndex)>` maps each eclass to its cheapest enode; patched when enode absorbed during rebuild
- **ActionCtx fields:** `cost_models: &'a Map<String, u64>`, `eclass_cost: &'a mut Map<Value, u64>`, `cost_select: &'a mut Map<Value, (TableId, RowIndex)>` added to ActionCtx
- **No worklist propagation** in this phase — bottom-up insert order handles common case; Phase 4 can extend
