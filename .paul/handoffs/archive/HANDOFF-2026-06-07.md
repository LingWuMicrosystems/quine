## Decisions Made

| Decision | Rationale | Impact |
|----------|-----------|--------|
| CSE adjustment is parent-side, not child-side | `OwnedBy(owner)` changes how parents account for child cost; child still picks cheapest enode independently | Cleaner separation in `enode_cost_with_cse` |
| Depth-first B&B (no BinaryHeap priority queue) | Sufficient for Phase 7 per design report §4.3 | Simpler, fewer allocations; best-bound-first deferred to Phase 9 |
| `Solution` struct lives in `relaxation.rs` | Used by both relaxation and solver; placing where both can import avoids circular deps | Clean dependency graph |
| `eclass_map` built on-the-fly in solver functions | Avoids modifying 07-01's `ExtractionDAG` struct | Respects boundaries, negligible overhead |
| `child_parents` dedup at DAG construction level | Same `(parent, enode)` pair shouldn't create a CSE edge; fix at data source | No changes needed in relaxation or solver |

## Gap Analysis

### #16: FixedDecision enum can't represent simultaneous Selected + OwnedBy
**Status:** DEFERRED to next task
**Problem:** In nested CSE scenarios, an eclass can both be `OwnedBy(parent_X)` and need `Selected(enode_ref)` forced. The enum only allows one variant. `BTreeMap::insert` overwrites the previous decision.
**Fix:** Refactor `FixedDecision` from enum to struct with two independent `Option` fields:
```rust
pub struct FixedDecision {
    pub selected: Option<(TableId, RowIndex)>,
    pub cse: Option<CseDecision>,  // OwnedBy(usize) | NotShared
}
```
`BTreeMap` insert becomes a merge operation.
**Files:** `quine-solver/src/relaxation.rs` (type + all match sites), `quine-solver/src/solver.rs` (branch_and_bound insertions)
**Effort:** ~15min

### #17: child_parents records duplicate (parent, enode) pairs
**Status:** FIXED in dag.rs
**Problem:** `Add(A, A)` pushed `(idx, enode_i)` twice for child A → false CSE edge → wasted B&B branching
**Fix:** Added `.contains()` guard before `.push()` in `build_extraction_dag()`
